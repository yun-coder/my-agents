# LangSmith 详细能力报告


## 1. 本地私有化部署详解

官方提供了详细的最佳实践指南，适用于对数据安全、合规性有严格要求的企业场景。一套生产级的自托管方案包含以下核心组件与配置：

### 1.1 部署方案架构

| 核心组件 | 配置要求 | 说明 |
| :--- | :--- | :--- |
| **核心服务** | 配置自动扩缩容（Autoscaling） | LangSmith Backend、Platform Backend、Queue Service 等服务需根据 CPU 使用率等负载指标，设置最小/最大副本数，以应对流量波动。 |
| **PostgreSQL** | 强烈推荐使用托管外部服务 | 作为主数据库，推荐使用 AWS RDS 等，避免在集群内自行管理。 |
| **Redis** | 强烈推荐使用托管外部服务 | 用于缓存等场景，推荐使用 AWS ElastiCache 等。 |
| **ClickHouse** | 高可用（HA）复制部署 + 对象存储卸载 | 用于存储和分析海量追踪数据。需使用 ZooKeeper 协调多副本，并建议启用对象存储（如 AWS S3）卸载大型跟踪数据，以降低数据库压力与成本。 |
| **运维监控** | 日志、指标、链路追踪全面覆盖 | 对自托管实例本身进行监控，配置关键告警（如高错误率、服务宕机、数据库连接池耗尽等）。 |

### 1.2 部署流程摘要
1. **环境准备**：基于 Kubernetes 集群。
2. **安装**：通过 Helm Chart 等工具进行初始安装。
3. **配置优化**：按序配置外部数据存储、核心服务扩缩容策略、运维监控与告警。

## 2. 监控维度与最小颗粒度

LangSmith 的监控体系分层且多维，可深入到单次 LLM 调用的每一个细节。

### 2.1 监控维度

| 监控层级 | 监控内容 | 典型应用场景 |
| :--- | :--- | :--- |
| **项目级（Project）** | 错误率、延迟、Token 消耗量、成本趋势等核心健康度指标（通过预置仪表盘呈现）。 | 全局把控应用整体状况。 |
| **跟踪级（Trace）** | 一次完整用户请求（如多步骤 Agent 任务）的聚合延迟、成本、Token 用量。 | 分析端到端性能与用户体验。 |
| **运行级（Run）** | 跟踪下每次 LLM 调用、工具调用等具体操作的计数、延迟、错误率，可按运行类型（如 `llm`、`tool`）筛选。 | 定位具体步骤的性能瓶颈或错误根源。 |

### 2.2 最小颗粒度
监控的最小颗粒度为 **单次“运行（Run）”级别**。你可以查看任何一次模型调用的：
*   完整输入、输出、中间状态
*   Token 消耗明细
*   延迟指标
*   错误信息堆栈

## 3. 监控接口参数与 Token 用量报告

### 3.1 详细监控参数
通过 SDK 或 API 收集的每次“运行”数据包含以下关键信息：

| 参数类别 | 字段示例 | 说明 |
| :--- | :--- | :--- |
| **基础元数据** | `trace_id`, `run_id`, `run_type`(如`llm`), `name`, `start_time`, `end_time`, `error` | 唯一标识与状态记录。 |
| **模型信息** | `ls_model_name`, `ls_provider` | 用于匹配成本计算。 |
| **Token 用量** | `input_tokens`, `output_tokens`, `total_tokens` | 基本 Token 计数。 |
| **Token 明细** | `input_token_details` (如`cache_read`, `audio`, `image`), `output_token_details` (如`reasoning`, `image`) | 精确到不同类型的 Token 消耗。 |
| **性能信息** | `latency` | 运行耗时。 |

### 3.2 模型 Token 用量报告
*   **成本计算**：根据内置或自定义的模型价格表，LangSmith 自动计算每次运行、每个跟踪乃至整个项目的成本。
*   **报告维度**：预置仪表盘可按 Token 类型（提示/完成）细分，展示总用量、每跟踪用量及成本。自定义仪表盘可构建更个性化的图表（如对比不同模型的 Token 消耗成本）。

## 4. 多个 Agent 的监听能力

**LangSmith 可对多个 Agent 进行统一或独立的监听。**

*   **独立监听**：为每个 Agent 创建独立的 **项目（Project）**，实现追踪数据、评估、仪表盘的完全隔离。
*   **统一监听**：创建 **自定义仪表盘** 并关联多个项目，在统一视图中同时监控多个 Agent 的核心指标（如总请求量、总成本）。
*   **多 Agent 交互场景**：LangSmith 的链路追踪能串联起不同 Agent 间的调用关系。对于多轮、多 Agent 的会话（Thread），也可在评估层面衡量其整体质量。

## 5. 能力边界总结

### 5.1 核心能力（它能做什么）
*   **全链路可观测性**：从最外层用户请求到最内层单个 LLM 调用，全程追踪与记录。
*   **多维监控与告警**：覆盖性能、成本、质量、资源等指标，支持自定义仪表盘和告警。
*   **自动化测试与评估**：支持离线测试集与在线生产数据评估，通过“LLM 作为评判者”等方式衡量模型表现。
*   **数据驱动优化**：通过分析生产数据，发现用户行为模式与高频失败原因，为模型和提示词迭代提供方向。
*   **灵活部署**：支持 SaaS 云服务与本地自托管部署。

### 5.2 能力边界（它不做什么）
*   **不提供 LLM 服务**：需自行接入 OpenAI、Anthropic 等模型提供商的 API。
*   **不是 Agent 框架**：需使用 LangChain 或其他框架构建应用，LangSmith 负责监控和评估。
*   **不是数据处理平台**：它不执行用户的数据清洗或计算任务，而是监控这些任务的执行过程。
