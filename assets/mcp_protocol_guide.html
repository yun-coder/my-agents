🔌 MCP 协议完全图解

Model Context Protocol — 让 AI 工具"即插即用"的统一标准

零、一句话理解 MCP

🍕 生活类比：MCP 就像 USB 接口。以前每个 AI 工具都要单独写驱动，现在只要插上 MCP 接口，AI 就能立刻识别并使用它。
📦 一句话：MCP = AI 模型 + 外部工具的"通用翻译官"。

一、MCP 只有三个角色（超简单！）

角色类比职责常见例子
Client 顾客点餐 发起请求，说"我需要某某功能" Claude Desktop, Cursor, VS Code 扩展
Transport 外卖骑手 负责把消息从 A 送到 B，不管内容 STDIO(本地进程), HTTP(网络), SSE(服务器推送)
Server 餐厅厨房 提供具体能力，执行后返回结果 文件读取器、GitHub API、数据库查询

二、一次完整通信的 6 步走

下面这张图展示了从"插上 USB"到"拿到答案"的完整过程。左边是 Client（AI），右边是 Server（工具），中间是来回传递的消息。

tools/list — Server 告诉 Client "我有什么"，Client 不用提前知道 • tools/call — Client 说"帮我做这个"，带上参数，Server 执行后返回结果 • 双向通信 — 不是一次性的，可以反复调用多个工具，直到 AI 觉得够了为止

三、Transport 三种传输方式（消息怎么送？）

Transport 只管"把消息送到"，不管"送的是什么"。就像快递公司，不管你是寄文件还是寄蛋糕，它只负责安全送达。

四、MCP 三大核心操作（能做什么？）

MCP 只有三种操作：让工具干活(T)、读取数据(R)、使用对话模板(P)。记住这三个字母就够了。

六、一条消息长什么样？（JSON-RPC 格式）

📨 生活类比：MCP 消息就像一封快递单。不管内容是文件还是搜索结果，信封格式永远一样：谁发的 → 给谁的 → 是什么类型 → 内容是什么

七、MCP vs 其他协议（为什么需要 MCP？）

特性MCPREST APIgRPCWebSocket
定位 AI 工具的标准接口 通用 HTTP 接口 高性能 RPC 实时双向通信
谁在用 Claude, ChatGPT, 本地 LLM 几乎所有 Web 服务 微服务之间 聊天室, 直播弹幕

上手难度 ⭐ 低 (即插即用) ⭐⭐ 中 ⭐⭐⭐ 高 ⭐⭐⭐ 高
一句话总结 "AI 的 USB 接口" "互联网的 HTTP" "微服务的 gRPC" "实时流的 WebSocket"
💡 核心价值：MCP 让 AI 模型不再"孤岛"。以前每接入一个新工具都要写专用代码，现在只要实现 MCP Server，任何支持 MCP 的 AI 都能立即使用你的工具。这就是 "一次实现，处处可用"。

九、速查卡

问题答案
MCP 是什么？AI 模型和外部工具之间的统一标准接口
谁发明的？Anthropic (Claude 的母公司)，2024 年发布
为什么要用它？避免每个 AI 工具都写专用集成代码，实现"一次实现，处处可用"
支持哪些语言？官方 SDK: Python, TypeScript；社区: Go, Rust, Java...
怎么学？1) 装 SDK → 2) 写 Server → 3) 用 Claude Desktop 连接 → 4) 完事!
生产能用吗？✅ 已成熟，Claude Desktop、Cursor、Windsurf 等都原生支持
⚠️ 安全提醒：MCP Server 拥有执行工具的能力，务必做好权限控制。不要信任不可信的 Server，就像不要随便安装来源不明的软件一样。
