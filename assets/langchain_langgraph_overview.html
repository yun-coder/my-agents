四、LangChain Chain vs LangGraph 对比

维度 LangChain Chains LangGraph StateGraph
数据流模型 线性管道（Runnable 链式调用） 有向图（节点 + 边 + 循环）
控制流 顺序执行 / 并行合并 条件路由 / 循环 / 分支 / 中断
状态管理 每次传递完整 dict 共享 TypedDict，增量更新
循环能力 ReAct Agent 内置简单循环 原生支持任意循环（图边回指）
检查点 无（或外部实现） 内建 Checkpointer，每步快照
中断/恢复 不支持 interrupt() + Command(resume)
可视化 无 graph.get_graph().draw_mermaid_png()
适用场景 RAG 流水线、简单 Agent、Prompt→LLM→Parse 多步 Agent、人工审核、复杂工作流
底层依赖 — 依赖 LangChain 组件（LLM/Tool/RAG）

典型代码结构：

# 1. LangChain 组件
llm = ChatOpenAI(model="gpt-4")
tools = [search_tool, calculator_tool]
retriever = vectorstore.as_retriever()

# 2. LangGraph 图构建
builder = StateGraph(AgentState)
builder.add_node("agent", agent_node)       # 使用 LangChain LLM
builder.add_node("tools", tool_node)         # 使用 LangChain Tools
builder.add_node("rag", rag_node)            # 使用 LangChain RAG
builder.set_entry_point("agent")
builder.add_conditional_edges("agent", router, {"tools": "tools", "rag": "rag", "end": END})
builder.add_edge("tools", "agent")
builder.add_edge("rag", "agent")

# 3. 编译 & 运行
graph = builder.compile(checkpointer=MemorySaver())
result = graph.invoke({"messages": [...]}, config={"configurable": {"thread_id": "123"}})
