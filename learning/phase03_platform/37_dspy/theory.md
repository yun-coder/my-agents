# 37. DSPy

## 学习目标

- 理解 DSPy 将提示词工程转化为可评估、可优化的程序。
- 会定义训练样本、评分函数和候选策略。
- 知道优化结果必须在独立验证集上检查。

## 核心概念

手工调整提示词适合早期探索。样本增加后，可以把任务描述、示例和评分函数放进优化流程。DSPy 的核心思路是：先声明程序结构和评价标准，再使用优化器搜索更好的提示策略。

一个最小优化闭环包括：

1. 准备有代表性的训练样本和验证样本。
2. 定义可计算的指标。
3. 生成或选择候选提示策略。
4. 在训练集上优化。
5. 在独立验证集上确认没有过拟合。

## 示例说明

`demo.py` 不依赖模型，使用关键词分类模拟候选策略评分。它展示的是“样本 + 指标 + 选择”的骨架。

## 运行

