# 29. 代码执行沙箱：E2B / Modal

## 学习目标

- 理解为什么执行模型生成代码必须使用隔离环境。
- 区分语法检查、策略检查和真正的操作系统级沙箱。
- 会为执行任务设置超时和最小权限。

## 核心概念

模型生成的代码属于不可信代码。生产环境不应直接在 API 服务进程中调用 `exec()`。基本控制包括：

| 控制 | 目的 |
| --- | --- |
| 隔离运行时 | 避免访问宿主机文件、进程和凭据 |
| 网络策略 | 禁止任意外连和数据泄露 |
| 超时与资源限额 | 防止死循环和资源耗尽 |
| 临时文件系统 | 任务结束后销毁数据 |
| 审计日志 | 记录提交代码、镜像、结果和错误 |

E2B 提供面向 Agent 的云沙箱。Modal 提供可按需执行的云运行环境。它们解决的是运行隔离问题，不等于代码一定安全。

## 示例说明

`demo.py` 使用 AST 做教学级策略检查，并通过独立 Python 子进程设置超时。它不是生产沙箱，只用于说明执行前后需要哪些边界。
