# 27. Prompt Injection Defense

## 学习目标

- 区分直接注入和间接注入。
- 理解外部网页、文件和工具结果都属于不可信数据。
- 会将“检测、隔离、最小权限、审批、监控”组合使用。

## 核心概念

Prompt Injection 的本质是：模型无法天然区分“应执行的指令”和“仅供分析的数据”。攻击者可能直接向聊天框输入恶意指令，也可能把恶意文本藏在网页、邮件或知识库文档中，让 Agent 在检索后读到。

防御措施应分层：

1. 明确标记外部内容为不可信数据。
2. 将模型可调用工具缩减到完成任务所需的最小集合。
3. 对发送邮件、修改数据、付款等动作要求确认。
4. 检测常见风险信号并记录日志。
5. 用攻击样本持续回归测试。

仅靠关键词检测不够，因为攻击方式可以改写和混淆。关键词扫描适合作为早期信号，不适合作为唯一防线。
