# 8.打造聊天机器人

# openai 模型规范
* 对具有不同角色的消息赋予不同的优先级
* developer  开发者	
* user  用户	
* assistant  助手
* developer 消息是由应用程序开发者提供的指令，其优先级高于 user 消息。	
* user 消息是由最终用户提供的指令，其优先级低于 developer 消息。	
* Messages generated by the model have the assistant role.
* 该模型生成的消息具有 assistant 角色。


* 使用 Markdown 和 XML 进行消息格式化
* 在构建消息时，您应该尽量将预期会在 API 请求中反复使用的内容放在提示信息的开头， 并将其作为传递给聊天完成或响应的 JSON 请求正文中的首批 API 参数之一。这样可以最大限度地利用提示缓存来节省成本和延迟。
* Few-shot learning  少样本学习
* Include relevant context information 包含相关的背景信息
  * To give the model access to proprietary data, or any other data outside the data set the model was trained on.
  * 使模型能够访问专有数据，或模型训练所用数据集之外的任何其他数据。
  * To constrain the model’s response to a specific set of resources that you have determined will be most beneficial.
  * 将模型的响应限制在您已确定的最有利的特定资源集上

* 在模型生成请求中添加额外相关上下文的技术有时被称为检索增强生成（RAG）
* 可以通过多种方式向提示中添加额外上下文，例如查询向量数据库并将返回的文本包含在提示中

* Planning for the context window 规划上下文窗口
