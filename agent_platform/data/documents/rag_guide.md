# RAG（检索增强生成）技术指南

## RAG 概述

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的技术架构。它通过在生成回答前从外部知识库中检索相关信息，让 LLM 能够基于权威资料回答问题，减少幻觉现象。

## RAG 工作流程

1. **文档加载**：从 PDF、Word、网页等来源加载文档。
2. **文本分块**：将长文档切分成适当大小的文本块（通常 300-1000 字符）。
3. **向量化**：使用 Embedding 模型将文本块转换为向量表示。
4. **向量存储**：将向量存入向量数据库（如 Chroma、Qdrant、Pinecone）。
5. **检索**：用户提问时，将问题向量化后在向量库中搜索最相似的文本块。
6. **上下文组装**：将检索结果与用户问题组装成 Prompt。
7. **生成回答**：LLM 基于组装后的 Prompt 生成最终回答。

## 提升 RAG 质量的技巧

### 1. 文档预处理
- 清理文档中的噪声和格式问题。
- 保留文档结构信息（标题、段落、表格）。
- 提取图片中的文字（OCR）。

### 2. 分块策略
- 语义分块优于固定长度分块。
- 保留适当的重叠（overlap）以维护上下文连贯性。
- 考虑文档的层级结构。

### 3. 检索优化
- 混合检索：结合语义搜索和关键词搜索（BM25）。
- 重排序（Reranking）：使用 CrossEncoder 对检索结果进行二次排序。
- 多路召回：从不同角度生成多个查询，合并召回结果。

### 4. Prompt 设计
- 明确指示模型"只能根据提供的资料回答"。
- 要求模型在回答中引用来源。
- 资料不充分时，允许模型说"不知道"。

## 向量数据库对比

| 数据库 | 类型 | 部署方式 | 特点 |
|---|---|---|---|
| Chroma | 开源 | 本地/Python | 轻量级，适合开发和中小规模应用 |
| Qdrant | 开源 | Docker/云 | 高性能，支持复杂过滤 |
| Pinecone | 商业 | SaaS | 全托管，免运维 |
| Weaviate | 开源 | Docker/云 | 支持混合搜索 |
| Milvus | 开源 | Docker/K8s | 大规模分布式，适合企业级应用 |
